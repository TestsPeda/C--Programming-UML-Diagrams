Datentypen & Variablen

Welcher Typ speichert welche Werte — und wie deklariert man eine Variable korrekt.

01Multiple ChoiceAuto

Welcher Datentyp eignet sich am besten für einen Geldbetrag wie die Futterkosten im Tierheim?

  • int
  • double
  • decimal
  • string
Lösung

decimal rechnet dezimal-genau (keine Rundungsfehler) und ist daher für Geldbeträge vorgesehen. Literale erhalten das Suffix m, z. B. decimal kosten = 2.7m;. Verwechslungdouble ist binäre Gleitkomma-Arithmetik → Rundungsfehler bei Geld.

02Multiple ChoiceAuto

Wie wird eine ganzzahlige Variable alter mit dem Wert 18 korrekt deklariert und initialisiert?

  • int alter == 18;
  • int alter = 18;
  • alter int = 18;
  • int alter := 18;
Lösung

Schema: <Typ> <Name> = <Wert>;. Das einfache = ist der Zuweisungsoperator. Verwechslung== ist der Vergleichsoperator, nicht für Zuweisungen.

03Multi-SelectAuto

Welche Aussagen über C#-Datentypen sind richtig? (mehrere möglich)

  • char speichert genau ein Zeichen und steht in einfachen Anführungszeichen: 'A'.
  • string ist eine Zeichenkette in doppelten Anführungszeichen: "Hallo".
  • bool kann die Werte true, false oder null annehmen.
  • int speichert ganze Zahlen ohne Nachkommastellen.
  • double ist für exakte Geldbeträge die beste Wahl.
Lösung

Richtig sind 1, 2 und 4. bool kennt nur true/false (kein null beim normalen bool), und für Geld nimmt man decimal, nicht double.

B

Typkonvertierung & Ein-/Ausgabe

Konsoleneingaben sind immer Text — und müssen vor dem Rechnen umgewandelt werden.

04Multiple ChoiceAuto

Welchen Datentyp liefert Console.ReadLine() zurück?

  • int
  • string
  • double
  • den jeweils passenden Typ automatisch
Lösung

Console.ReadLine() liefert immer einen string. Vor dem Rechnen muss man explizit konvertieren, z. B. int zahl = Convert.ToInt32(Console.ReadLine());.

05Multiple ChoiceAuto

Welche Umwandlung ist eine implizite Typkonvertierung (passiert automatisch, ohne Datenverlust)?

  • int → double
  • double → int
  • string → int
  • decimal → int
Lösung

Von int nach double geht implizit, weil der größere Typ jeden int-Wert aufnehmen kann (Erweiterung). Umgekehrt (Verkleinerung) droht Datenverlust → explizit per Cast (int) oder Convert.ToInt32(...). string → int ist nie implizit (Parsen nötig).

06Rechnenmanuell

Gegeben: double futtermenge = gewicht * 0.015; mit gewicht = 20.0 (kg). Wie viel Futter ergibt sich, und wie würden Sie das Ergebnis auf 2 Nachkommastellen gerundet ausgeben?

Musterlösung
futtermenge = 20.0 * 0.015 = 0.3 // kg

Gerundete Ausgabe:

Console.WriteLine($"Futter: {Math.Round(futtermenge, 2)} kg");

Math.Round(wert, 2) rundet kaufmännisch auf 2 Stellen. Mit String-Interpolation ($"...") werden Variablen in { } direkt eingesetzt.

Operatoren

Arithmetik, Vergleich und Logik — inklusive der typischen Stolperfalle Ganzzahldivision.

07Multiple ChoiceAuto

Was ergibt der Ausdruck 7 / 2 in C#, wenn beide Operanden int sind?

  • 3.5
  • 3
  • 4
  • Fehler beim Kompilieren
Lösung

Bei zwei int-Operanden führt C# eine Ganzzahldivision aus: Die Nachkommastellen werden abgeschnitten (nicht gerundet) → 3. Für 3.5 muss mindestens ein Operand double sein, z. B. 7.0 / 2.

08Rechnenmanuell

Bestimmen Sie die Ergebnisse: 17 % 5, 17 / 5 (int) und den booleschen Wert von (17 % 5 == 2) && (17 > 10).

Musterlösung
17 % 5 = 2 // Modulo: Rest der Division 17 / 5 = 3 // Ganzzahldivision: 5 passt 3-mal in 17 (17 % 5 == 2) && (17 > 10) = (2 == 2) && (17 > 10) = true && true = true

Der Modulo-Operator % liefert den Rest. && ist nur true, wenn beide Seiten true sind.

09Multiple ChoiceAuto

Mit welchem Ausdruck prüfen Sie, ob eine Zahl z ein Vielfaches von 3 ist?

  • z / 3 == 0
  • z % 3 == 0
  • z % 3 == 1
  • z * 3 == 0
Lösung

Teilbarkeit prüft man mit dem Rest: z % 3 == 0 bedeutet „bei Division durch 3 bleibt kein Rest" → Vielfaches von 3.

D

Verzweigungen & Schleifen

if/switch steuern Entscheidungen, Schleifen wiederholen Aktionen.

10Multiple ChoiceAuto

In einer switch-Anweisung — wozu dient das Schlüsselwort break; am Ende eines case?

  • Es beendet das gesamte Programm.
  • Es verlässt den switch-Block, damit nicht in den nächsten case „durchgefallen" wird.
  • Es springt zurück zum Schleifenanfang.
  • Es ist optional und hat keine Wirkung.
Lösung

break; beendet den switch nach dem passenden case. Ohne break verlangt C# in der Regel ein anderes Sprungziel — fehlt es, gibt es einen Compilerfehler. default: fängt alle übrigen Fälle ab.

11Multiple ChoiceAuto

Welche Schleife ist die richtige Wahl, wenn die Anzahl der Wiederholungen vorab feststeht (z. B. „zähle von 1 bis 100")?

  • for-Schleife
  • while-Schleife
  • do-while-Schleife
  • switch
Lösung

Die for-Schleife bündelt Startwert, Bedingung und Schrittweite in einer Zeile — ideal bei bekannter Wiederholungszahl: for (int i = 1; i <= 100; i++). while passt besser, wenn die Abbruchbedingung von Eingaben abhängt (z. B. „bis richtig geraten").

12Multiple ChoiceAuto

Was ist der Unterschied zwischen while und do-while?

  • Es gibt keinen Unterschied.
  • do-while prüft die Bedingung erst am Ende → der Schleifenkörper läuft mindestens einmal.
  • while läuft immer mindestens einmal, do-while nie.
  • do-while kann keine Bedingung enthalten.
Lösung

while (Bedingung) { ... } prüft vor dem ersten Durchlauf (kann also 0-mal laufen). do { ... } while (Bedingung); prüft nach dem Durchlauf → mindestens 1 Durchlauf. Praktisch z. B. für Menüs.

13Szenariomanuell

Ein Menü soll mit switch und einer while-Schleife so lange angezeigt werden, bis der Benutzer „Beenden" wählt. Skizzieren Sie die Programmstruktur (Pseudocode/Stichpunkte).

Musterlösung
bool beenden = false; while (!beenden) { // Menü ausgeben (1) Summe 1..100, (2) Beenden // Eingabe lesen + nach int konvertieren switch (auswahl) { case 1: // Summe 1..100 berechnen (for-Schleife) und ausgeben break; case 2: beenden = true; break; default: // Fehlermeldung "ungültige Option" break; } }

Kernidee: Die Schleife wiederholt das Menü, switch verzweigt nach Auswahl, eine bool-Steuervariable (oder break/return) beendet die Schleife.

14ProgrammierenSelbsttest

Zahlenraten. Schreiben Sie ein C#-Programm: Das Programm wählt eine Zufallszahl zwischen 1 und 20. Der Benutzer rät so lange, bis er richtig liegt; nach jedem Tipp gibt das Programm „zu hoch" oder „zu niedrig" aus. Am Ende wird die Anzahl der Versuche gemeldet.

Tipp: Code oben eintippen, dann „KI-Bewertungs-Prompt kopieren" → in ChatGPT/Claude einfügen für Feedback. Die Seite selbst führt keinen Code aus.

Musterlösung 
using System;

int zahl = new Random().Next(1, 21);  // 1..20 (21 exklusiv)
int versuche = 0;
int tipp;

Console.WriteLine("Errate die Zahl zwischen 1 und 20!");
do
{
    Console.Write("Dein Tipp: ");
    tipp = Convert.ToInt32(Console.ReadLine());
    versuche++;

    if (tipp < zahl)
        Console.WriteLine("Die Zahl ist zu niedrig.");
    else if (tipp > zahl)
        Console.WriteLine("Die Zahl ist zu hoch.");
}
while (tipp != zahl);

Console.WriteLine($"Richtig! Du hast die Zahl in {versuche} Versuchen erraten.");
Bewertungs-Checkliste
  • Random().Next(1, 21) → Zufallszahl 1–20 (Obergrenze exklusiv)
  • Schleife läuft, bis tipp == zahl
  • Eingabe gelesen und nach int konvertiert
  • Feedback „zu hoch" / „zu niedrig" korrekt
  • Versuchszähler hochgezählt und am Ende ausgegeben

15Multiple ChoiceAuto

Was passiert bei Convert.ToInt32("abc") zur Laufzeit?

  • Es ergibt 0.
  • Es wird eine FormatException geworfen (Programmabsturz, wenn nicht abgefangen).
  • Es ergibt null.
  • Der Text wird Zeichen für Zeichen addiert.
Lösung

Nicht-numerischer Text lässt sich nicht konvertieren → FormatException. Robuster ist int.TryParse(text, out int wert): liefert true/false statt abzustürzen.
