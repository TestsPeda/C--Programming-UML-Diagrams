01Multiple ChoiceAuto

Womit wird eine einzelne Aktion in einem Aktivitätsdiagramm dargestellt?

  • mit einer Raute
  • mit einem abgerundeten Rechteck
  • mit einem ausgefüllten Kreis
  • mit einem dicken Balken
Lösung

Die Aktion — das zentrale Element — ist ein abgerundetes Rechteck mit dem Tätigkeitsnamen, z. B. „Summe berechnen".

02Multiple ChoiceAuto

Wofür steht der ausgefüllte Kreis mit Ring (◉)?

  • Startpunkt
  • Endpunkt der Aktivität
  • Entscheidungsknoten
  • eine parallele Gabelung
Lösung

Der ausgefüllte Kreis ist der Startpunkt; der ausgefüllte Kreis mit Ring ist der Endpunkt der Aktivität. Das durchgestrichene Kreissymbol ⊗ ist das Ende eines einzelnen (abzweigenden) Steuerungsflusses.

03Multiple ChoiceAuto

Ein Entscheidungsknoten (Raute) hat …

  • einen Eingang und zwei oder mehr Ausgänge, die an Bedingungen geknüpft sind.
  • immer genau einen Eingang und einen Ausgang.
  • mehrere Eingänge, die parallel ausgeführt werden.
  • keine Bedingungen.
Lösung

Der Entscheidungsknoten verzweigt einen Eingang in mehrere Ausgänge. Jeder Ausgangspfad trägt eine Bedingung in eckigen Klammern, z. B. [neuer Kunde] / [bestehender Kunde]. Es wird stets nur ein Pfad genommen.

04Multi-SelectAuto

Welche Aussagen über Aktivitätsdiagramme sind richtig? (mehrere möglich)

  • Sie modellieren dynamische Abläufe (Reihenfolge von Aktionen).
  • Verantwortungsbereiche (Swimlanes) zeigen, wer für welche Aktion zuständig ist.
  • Jede Aktivität darf nur genau einen Endpunkt haben.
  • Eine Gabelung teilt den Fluss in parallele Teilflüsse auf.
  • Bedingungen am Entscheidungsknoten stehen in runden Klammern.
Lösung

Richtig: 1, 2, 4. Eine Aktivität darf mehrere End-/Flussende-Punkte haben, und Bedingungen stehen in eckigen Klammern [ ].

05Multiple ChoiceAuto

Was ist der Unterschied zwischen Entscheidung (Verzweigung) und Gabelung?

  • Es gibt keinen Unterschied.
  • Bei der Entscheidung wird ein Pfad (je Bedingung) genommen; bei der Gabelung laufen alle Teilflüsse parallel weiter.
  • Die Gabelung kennt Bedingungen, die Entscheidung nicht.
  • Die Entscheidung beendet die Aktivität.
Lösung

Entscheidung (Raute) = entweder/oder, gesteuert durch Bedingungen. Gabelung (Balken) = sowohl als auch, die Teilflüsse werden parallel abgearbeitet und später durch eine Synchronisation/Vereinigung wieder zusammengeführt.

07Multiple ChoiceAuto

Wie viele Startpunkte muss eine Aktivität mindestens haben?

  • keinen
  • mindestens einen
  • genau zwei
  • so viele wie Aktionen
Lösung

Jede Aktivität hat mindestens einen Start- und mindestens einen Endpunkt. Der Ablauf beginnt am ausgefüllten Kreis.

08Modellierenmanuell

Modellieren Sie als Aktivitätsdiagramm: „Zahl einlesen. Ist die Zahl ungleich 0, den Kehrwert berechnen, sonst eine Fehlermeldung ausgeben." Beschreiben Sie die Elemente und ihre Reihenfolge.

Musterlösung
  1. Startpunkt (●)
  2. Aktion „Zahl einlesen"
  3. Entscheidungsknoten mit Bedingung [Zahl ungleich 0] bzw. [Zahl gleich 0]
  4. Pfad [ungleich 0] → Aktion „Kehrwert berechnen"
  5. Pfad [gleich 0] → Aktion „Fehlermeldung ausgeben"
  6. Vereinigung (Raute) beider Pfade
  7. Endpunkt (◉)

Das entspricht 1:1 einer if-else-Verzweigung im Code.

09ProgrammierenSelbsttest

Diagramm → Code. Setzen Sie den Entscheidungsteil des Ticketkaufs in C# um: Frage den Benutzer, ob er bereits ein Konto hat (j/n). Bei „j" gib „Anmelden …" aus, sonst „Registrieren …". Danach in beiden Fällen „Zahlungsdaten eingeben" ausgeben (Vereinigung der Pfade).

Erweiterung (schwerer) — Geldautomat aus Diagramm 2: Setzen Sie das vollständige Diagramm mit Schleife um: PIN abfragen, bei falschem PIN erneut fragen, jedoch maximal 3 Versuche. Nach dem 3. Fehlversuch „Karte einziehen" ausgeben und abbrechen; bei korrektem PIN „Betrag wählen" und „Geld ausgeben".

Tipp: Code oben eintippen, dann „KI-Bewertungs-Prompt kopieren" → in ChatGPT/Claude einfügen für Feedback. Die Seite selbst führt keinen Code aus.

Musterlösung 
using System;

Console.Write("Konto vorhanden? (j/n): ");
string antwort = Console.ReadLine();

// Entscheidungsknoten -> if-else
if (antwort == "j")
    Console.WriteLine("Anmelden ...");      // [ja]-Pfad
else
    Console.WriteLine("Registrieren ...");  // [nein]-Pfad

// Vereinigung: gemeinsame Folgeaktion in BEIDEN Fällen
Console.WriteLine("Zahlungsdaten eingeben");

Erweiterung (schwerer) — Geldautomat mit Schleife (Diagramm 2):

using System;

string echterPin = "1234";
int versuche = 0;
bool erfolg = false;

// Schleife im Code = Rückführung (Schleife) im Diagramm
while (versuche < 3)
{
    Console.Write("PIN eingeben: ");
    string pin = Console.ReadLine();
    versuche++;

    if (pin == echterPin)        // Entscheidung: PIN korrekt?
    {
        erfolg = true;
        break;                   // raus aus der Schleife -> weiter im Hauptfluss
    }
    Console.WriteLine("PIN falsch.");
}

if (erfolg)
{
    Console.WriteLine("Betrag wählen");
    Console.WriteLine("Geld ausgeben");   // (parallel: + Beleg drucken)
}
else
{
    Console.WriteLine("Karte einziehen"); // 3. Fehlversuch -> Abbruch (Flussende)
}
Bewertungs-Checkliste
  • Eingabe j/n eingelesen
  • if-else bildet den Entscheidungsknoten ab
  • „Anmelden" bei j, „Registrieren" bei n
  • „Zahlungsdaten eingeben" steht nach dem if-else (Vereinigung) → läuft in beiden Pfaden
  • Erweiterung: Schleife mit Zähler, max. 3 Versuche (= Schleife im Diagramm)
  • Erweiterung: Erfolg → „Betrag wählen"/„Geld ausgeben"; sonst „Karte einziehen" (Flussende)
10Multiple ChoiceAuto

Eine Vereinigung (Merge) nach einer Verzweigung entspricht im Code …

  • der gemeinsamen Anweisung nach dem if-else-Block.
  • einer zusätzlichen Schleife.
  • dem Ende des Programms.
  • einem zweiten if innerhalb des ersten.
Lösung

Nach der Vereinigung läuft ein gemeinsamer Fluss weiter — im Code die erste Anweisung, die hinter dem if-else steht und damit unabhängig vom gewählten Pfad ausgeführt wird.

11Zeichnen (Papier)manuell

Onlineshop-Bestellvorgang mit Verantwortungsbereichen. Zeichnen Sie das Aktivitätsdiagramm mit zwei Swimlanes Kunde und Onlineshop nach dieser Beschreibung:

  • Der Kunde wählt Artikel aus und geht anschließend zur Kasse.
  • Ein registrierter Kunde meldet sich an (einloggen); andernfalls bestellt er als Gast und gibt seine Adresse ein.
  • Danach wählt der Kunde die Zahlungsart und erteilt den Kaufauftrag.
  • Der Onlineshop bestätigt die Bestellung, erstellt gleichzeitig die Rechnung und packt die Ware (parallel).
  • Anschließend versendet der Onlineshop die Ware.
Musterlösung Prüf-Checkliste (auf deinem Blatt)
  • genau ein Startpunkt ● und ein Endpunkt ◉
  • zwei Verantwortungsbereiche (senkrechte Linie, je Bereich benannt)
  • Entscheidungsraute mit Bedingungen [ja]/[nein] + Vereinigung
  • Gabelung (Balken) → Rechnung & Ware packen parallel → Synchronisation (Balken)
  • jede Aktion im richtigen Bereich (Kaufauftrag = Kunde; Bestätigen/Packen/Versenden = Onlineshop)
12Zeichnen (Papier)manuell

Geldautomat (mit Schleife & Abbruch). Zeichnen Sie das Aktivitätsdiagramm nach dieser Beschreibung:

  • Karte einführen, dann PIN eingeben.
  • Ist der PIN falsch, wird erneut nach dem PIN gefragt — aber höchstens 3 Versuche.
  • Nach dem 3. Fehlversuch wird die Karte eingezogen und der Vorgang abgebrochen (Flussende).
  • Bei korrektem PIN wird der Betrag gewählt; danach werden gleichzeitig Geld ausgegeben und ein Beleg gedruckt.
  • Zum Schluss wird die Karte ausgegeben.
Musterlösung Prüf-Checkliste (auf deinem Blatt)
  • Schleife: Pfad „PIN falsch & noch Versuche" führt zurück zu „PIN eingeben"
  • verschachtelte Entscheidung: „PIN korrekt?" und „3. Versuch?"
  • Flussende ⊗ nach „Karte einziehen" (Abbruch, kein normales Ende)
  • Gabelung Geld ausgeben ‖ Beleg drucken → Synchronisation
  • genau ein Startpunkt ● und ein regulärer Endpunkt ◉ (Karte ausgeben)

13Multiple ChoiceAuto

Was kennzeichnet ein Verantwortungsbereich (Swimlane) im Aktivitätsdiagramm?

  • eine parallele Abarbeitung von Aktionen
  • eine vertikale/horizontale Einteilung, die zeigt, wer (welche Rolle) eine Aktion ausführt
  • das Ende eines Steuerungsflusses
  • eine Bedingung am Entscheidungsknoten
Lösung

Swimlanes teilen das Diagramm in benannte Bereiche (z. B. „Sachbearbeiter" / „Lagermitarbeiter"). Jede Aktion liegt in dem Bereich, dessen Rolle für sie verantwortlich ist.

14Multiple ChoiceAuto

Aktivitätsdiagramm, PAP (Programmablaufplan) und Struktogramm haben gemeinsam, dass sie …

  • Abläufe/Algorithmen grafisch darstellen (nur mit unterschiedlicher Notation).
  • ausschließlich Klassen und ihre Attribute zeigen.
  • nur in C# funktionieren.
  • keine Verzweigungen darstellen können.
Lösung

Alle drei beschreiben Abläufe grafisch. Das Struktogramm (Nassi-Shneiderman) nutzt verschachtelte Blöcke, der PAP Sinnbilder mit Pfeilen, das Aktivitätsdiagramm die UML-Notation — inhaltlich lassen sie sich ineinander überführen.
