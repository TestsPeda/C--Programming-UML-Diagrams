Grundlagen

zum Test öffnen →

Variable

Benannter Speicherplatz: Typ name = Wert; — Zuweisung mit einfachem =.

int / double / decimal

int = ganze Zahlen, double = Gleitkomma, decimal = exakt für Geld (Suffix m).

string / char / bool

string in "...", char ein Zeichen in '.', bool nur true/false.

Ein-/Ausgabe

Console.ReadLine() liefert immer string; Ausgabe mit Console.WriteLine($"...{x}").

Typkonvertierung

Erweiternd (int→double) implizit; verkleinernd explizit per Convert.ToInt32(...) / Cast.

Ganzzahldivision & %

7/2 = 3 (Rest fällt weg), 7%2 = 1 (Rest). Teilbarkeit: z % n == 0.

Verzweigung

if/else if/else für Bereiche; switch-case + break für feste Einzelwerte.

Schleifen

for bei bekannter Anzahl, while bei eingabeabhängigem Abbruch, do-while mind. 1×.

02

Arrays

zum Test öffnen →

Array (1D)

Reihe gleichartiger Werte: int[] a = new int[6];. Zugriff über Index a[i].

0-basiert

Erstes Element [0], letztes [Length - 1]. a[Length] → IndexOutOfRangeException.

Length

a.Length = Anzahl Elemente. Durchlaufen: for (int i=0; i<a.Length; i++).

Array (2D)

Tabelle: string[,] m = new string[3,4];. Zugriff m[zeile, spalte] (Komma!).

GetLength

GetLength(0) = Zeilen, GetLength(1) = Spalten; Length = Zeilen × Spalten gesamt.

Indexumrechnung

Menschliche Eingabe 1-basiert → Array 0-basiert: m[zeile-1, spalte-1] (Chaos-Lager).

Verschachtelte Schleife

Alle Felder eines 2D-Arrays: äußere Schleife Zeilen, innere Spalten.

03

UML-Aktivitätsdiagramm

zum Test öffnen →

Aktion

Abgerundetes Rechteck = eine Tätigkeit/Funktion des Systems.

Steuerungsfluss

Pfeil zwischen Aktionen — gibt die Reihenfolge vor.

Start / Ende

● Startpunkt, ◉ Endpunkt der Aktivität, ⊗ Ende eines einzelnen Flusses.

Entscheidung

Raute: ein Eingang, mehrere bedingte Ausgänge [Bedingung] — nur ein Pfad.

Vereinigung (Merge)

Raute, die verzweigte Pfade wieder zu einem Fluss zusammenführt → im Code: Anweisung nach if-else.

Gabelung

Dicker Balken: teilt den Fluss in parallele Teilflüsse (sowohl als auch).

Verantwortungsbereich

Swimlane: zeigt, wer (welche Rolle) für eine Aktion zuständig ist.
